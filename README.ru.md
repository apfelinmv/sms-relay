# SMS Relay

[English version](README.md)

SMS Relay — небольшое Android-приложение с открытым исходным кодом и сервер на чистом
Python, которые пересылают входящие SMS выбранным пользователям Telegram. Для SIM 1 и
SIM 2 задаются отдельные списки получателей. Пользователь Telegram начинает получать SMS
только после подтверждения собственного номера, уже добавленного в разрешённый список.

Проект рассчитан на недорогой Android-телефон, постоянно находящийся дома. В приложении
нет рекламы, аналитики, циклического опроса сервера, облачных SDK и сторонних Android-
библиотек.

## Как устроена система

```text
Мобильная сеть -> событие SMS + изменение системного inbox -> локальная очередь
               -> HTTPS + TLS pin -> личный сервер -> SQLite -> Telegram
```

1. Android передаёт событие `SMS_RECEIVED` классу `SmsReceiver`. Foreground guard также
   подписан на изменения системного inbox; это событийная подписка без опроса по таймеру.
2. Части длинной SMS объединяются по отправителю. Если прошивка передала пустое или
   непригодное событие, `InboxRecovery` читает только новые свежие строки inbox. Затем
   приложение определяет физический слот SIM и синхронно сохраняет SMS в закрытую очередь.
3. У каждой установки есть стабильный псевдонимный ID и понятное имя устройства. Android
   `JobScheduler` ждёт появления сети и отправляет очередь на сервер по HTTPS.
   Токен бота используется как ключ авторизации. SHA-256-отпечаток сертификата позволяет
   безопасно использовать самоподписанный сертификат для IP-адреса.
4. Сервер проверяет токен и данные, записывает SMS и отдельные задания для получателей в
   SQLite, фиксирует транзакцию и только после этого подтверждает приём телефону.
5. Серверный поток отправляет SMS через Telegram Bot API. Ошибки повторяются с растущей
   задержкой независимо для каждого получателя.
6. Когда SMS получили все адресаты, сервер очищает отправителя, текст и время сообщения.
   Детерминированный ID и служебные данные доставки хранятся до семи дней для защиты от
   повторной отправки.

Foreground guard не опрашивает сеть и не выполняет периодическую работу. Он показывает
тихое постоянное уведомление и уменьшает вероятность выгрузки процесса. Он **не обходит**
ограничения производителя: на Tecno HiOS обязательно выполнить настройки ниже.

## Какие файлы находятся в репозитории

- `app/src/main/java/.../SmsReceiver.java` — разбирает SMS и определяет SIM.
- `InboxRecovery.java` — восстанавливает только новые свежие строки системного SMS inbox.
- `SmsQueue.java` — синхронно хранит до 100 неотправленных SMS в закрытых данных приложения.
- `DeliveryJobService.java` и `DeliveryScheduler.java` — повторы отправки с учётом сети.
- `RelaySender.java` — HTTPS-клиент и проверка SHA-256-отпечатка сертификата.
- `RelayKeepAliveService.java` — foreground guard, наблюдатель inbox и уведомление.
- `MainActivity.java` и `SettingsStore.java` — экран и локальное хранение настроек.
- `server/sms_relay.py` — HTTP API, SQLite, подтверждение получателей и доставка в Telegram.
- `server/install.sh` — установка на Debian/Ubuntu.
- `server/nginx-sms-relay.conf` — TLS-прокси для `/sms`, `/configure` и `/health`.
- `server/test_sms_relay.py` — регрессионные тесты серверной части.

## Что потребуется

- Android 6.0 или новее с поддержкой SMS.
- Telegram-бот, созданный через `@BotFather`.
- Публичный сервер Debian/Ubuntu с Python 3, Nginx, systemd, статическим IP и доступным с
  телефона TCP-портом 443.
- Домен не нужен: установщик создаёт самоподписанный сертификат для IP-адреса.

Одна установка сервера хранит один токен бота и может обслуживать несколько Android-
телефонов. У каждого телефона независимые маршруты SIM 1 и SIM 2, поэтому **Save and sync**
изменяет только этот телефон. Все устройства намеренно используют одного Telegram-бота,
но могут пересылать SMS разным подтверждённым пользователям. Для другого владельца или
бота нужен отдельный экземпляр сервера.

## 1. Установка сервера

На сервере клонируйте репозиторий и запустите установщик от root:

```sh
sudo apt update
sudo apt install -y git nginx openssl python3
git clone https://github.com/apfelinmv/sms-relay.git
cd sms-relay
sudo ./server/install.sh ВАШ_ПУБЛИЧНЫЙ_IP
```

Скрипт создаёт отдельного системного пользователя `sms-relay` и устанавливает:

- `/opt/sms-relay/sms_relay.py`
- `/etc/systemd/system/sms-relay.service`
- `/etc/nginx/sites-available/sms-relay`
- `/etc/sms-relay/tls.crt` и `/etc/sms-relay/tls.key`
- `/var/lib/sms-relay/messages.sqlite3`

Сохраните SHA-256-отпечаток сертификата, напечатанный в конце. В Android его можно вводить
с двоеточиями или без них. Проверьте сервер:

```sh
systemctl status sms-relay --no-pager
curl -k https://ВАШ_ПУБЛИЧНЫЙ_IP/health
```

Правильный ответ: `{"ok":true}`. Если на сервере уже есть сайты или занят порт 443, перед
установкой проверьте `server/nginx-sms-relay.conf`: это конфигурация `default_server`, и её
нужно объединить с существующей конфигурацией Nginx.

Первая корректная команда **Save and sync** привязывает сервер к токену бота. Запрос с
другим токеном будет отклонён. Для намеренной замены бота остановите сервис, сделайте
резервную копию базы и удалите строку `config` и старых получателей из SQLite.

## 2. Создание Telegram-бота

1. Откройте официальный аккаунт `@BotFather`.
2. Отправьте `/newbot`, выберите имя и сохраните выданный токен.
3. Считайте токен паролем. Не публикуйте его в Git и не встраивайте в APK.

Бот не может первым написать пользователю. Каждый получатель должен один раз выполнить
подтверждение из раздела 4.

## 3. Установка и настройка Android

Скачайте универсальный APK из
[последнего GitHub Release](https://github.com/apfelinmv/sms-relay/releases/latest), перенесите
его на телефон, откройте и разрешите установку из этого источника. Для готового APK
Android Studio не нужна.

Откройте SMS Relay и заполните:

1. **Device name:** короткая подпись источника в Telegram, например `Домашний Tecno` или
   `Samsung мамы`.
2. **Server URL:** `https://ВАШ_ПУБЛИЧНЫЙ_IP`.
3. **Telegram bot token:** полный токен от `@BotFather`.
4. **SIM 1 recipients:** номера разрешённых Telegram-аккаунтов по одному в строке, в полном
   международном формате, например `+77001234567`.
5. **SIM 2 recipients:** отдельный список для SMS второй SIM. Если она не используется,
   оставьте поле пустым.
6. **TLS certificate SHA-256:** отпечаток, напечатанный `install.sh`. Оставлять пустым можно
   только при сертификате, которому Android уже доверяет.

Нажмите **Save and sync**, выдайте оба запрошенных разрешения на SMS и разрешение на
уведомления и не отключайте
уведомление **SMS Relay active**. После подтверждения получателей используйте
**Test SIM 1** и **Test SIM 2**.

## 4. Подтверждение каждого получателя в Telegram

1. Получатель открывает созданного бота и отправляет `/start`.
2. Бот показывает кнопку **Share my phone number**.
3. Получатель нажимает кнопку и делится собственным контактом Telegram.
4. Сервер нормализует номер и сравнивает его со списками для SIM.
5. Только при точном совпадении бот отвечает **Access granted**.

Пересланный чужой контакт отклоняется: Telegram user ID контакта должен совпадать с
отправителем. Если удалить номер в Android и нажать **Save and sync**, доступ получателя
будет отозван для этого устройства. Если тот же номер разрешён другим телефоном, его доступ
сохранится.

### Несколько телефонов на одном сервере

- На каждый дополнительный телефон устанавливайте версию 1.2.0 или новее.
- Используйте одинаковые адрес сервера, TLS-отпечаток и токен бота.
- Задайте каждому телефону отдельное понятное имя.
- Номера Telegram-получателей настраиваются независимо на каждом телефоне.
- Новая установка получает стабильный псевдонимный ID из идентификатора Android, доступного
  только этому приложению. Повторная установка на тот же телефон обычно вернётся к прежнему
  серверному маршруту.
- При обновлении уже настроенного телефона с 1.1.0 старые маршруты переносятся один раз и не
  меняют настройки новых телефонов.

## 5. Обязательные настройки телефона

Производители Android могут блокировать SMS-события, даже когда стандартный экран батареи
показывает «Без ограничений». Включите все доступные пункты:

1. Разрешите оба запрошенных доступа к **SMS** и **Уведомления**.
2. Разрешите **Автозапуск** SMS Relay.
3. Для батареи выберите **Без ограничений** / **Не оптимизировать**.
4. Разрешите фоновый мобильный интернет и Wi-Fi.
5. Отключите **Приостанавливать работу в неактивный период**.
6. Не отключайте постоянное уведомление **SMS Relay active**.
7. После перезагрузки или обновления прошивки один раз откройте SMS Relay. В статусе должно
   быть `Configured: yes`, `SMS permission: granted`, `Foreground guard: active`.

### Tecno HiOS: критически важная дополнительная настройка

На проверенной прошивке Tecno HiOS автозапуска, обычного Android-режима «Без ограничений»
и блокировки карточки в недавних приложениях оказалось недостаточно. HiOS записывал
`Hiber/broadcast restricted` и не передавал SMS приложению.

Откройте:

**Настройки -> Батарея / Battery Lab -> Управление энергосбережением для приложений -> SMS Relay -> Нет ограничений**

Выберите именно **Нет ограничений**, а не **Интеллектуальное управление**. Если экран скрыт,
подключите ADB и откройте его напрямую:

```sh
adb shell am start -a com.transsion.batterylab.app_saving
```

Затем выберите **SMS Relay -> Нет ограничений**. Блокировку карточки SMS Relay в недавних
приложениях можно оставить как дополнительную меру, но она не заменяет Battery Lab.

Проверьте настоящей входящей SMS после нескольких минут с заблокированным экраном. USSD-
запрос баланса не является SMS и не проверяет событие `SMS_RECEIVED`.

## Пределы надёжности

- Локальная очередь хранит 100 сообщений. Если неотправленных SMS станет больше, удаляется
  самая старая. Максимальная длина текста — 3500 символов.
- SMS записывается локально до запуска сетевой отправки. Перезагрузка и временное отсутствие
  интернета не очищают очередь.
- Android и сервер выполняют повторы независимо. Сервер начинает с 30 секунд и увеличивает
  задержку максимум до одного часа.
- Детерминированные ID делают повторную загрузку безопасной и не создают дубли.
- Приложение не становится стандартным SMS-клиентом и не удаляет SMS с телефона.
- Если обычное событие `SMS_RECEIVED` пустое или непригодное, но SMS появилась в системном
  inbox Android, работающий наблюдатель восстановит её автоматически. Открытие приложения
  и запуск после перезагрузки также проверяют свежие строки. SMS, которую Android вообще не
  записал в inbox, восстановить невозможно, поэтому настройки батареи остаются обязательными.
- USSD-окна, push-уведомления, сообщения мессенджеров и экстренные cell broadcast не являются
  обычными SMS и не пересылаются.

## Модель безопасности

- Разрешён только HTTPS. Самоподписанный IP-сертификат проверяется по SHA-256-отпечатку.
- Токен бота является ключом API и хранится в закрытых данных Android и серверной базе
  SQLite. В публичном APK и репозитории его нет.
- У сервера нет API для чтения сохранённых SMS. Логи доступа Nginx для relay отключены.
- После доставки всем получателям содержимое SMS очищается на сервере. До доставки оно
  сохраняется, чтобы работали повторы.
- Android backup и извлечение данных отключены для настроек и очереди.
- Разрешение `READ_SMS` используется только для чтения свежих строк inbox при восстановлении.
  Приложение не имеет функции выгрузки истории и не отправляет старые SMS.
- Чаты Telegram-бота используют облачное шифрование Telegram, а не end-to-end. Telegram,
  украденный токен, взломанный сервер или аккаунт получателя могут раскрыть SMS. При утечке
  токена отзовите его и заново настройте сервер.
- Пересылка OTP увеличивает число систем, имеющих доступ к коду. Используйте отдельный
  сервер, SSH-ключи, обновления ОС и минимальный список получателей.

## Как содержать домашний телефон

- Держите его в прохладном проветриваемом месте, без солнца и мягких поверхностей.
- Включите фирменный лимит заряда или защиту батареи. Если их нет, таймер или умная розетка
  помогут не держать аккумулятор постоянно на 100%.
- Обеспечьте стабильные Wi-Fi и мобильный сигнал: SIM должна оставаться в сети для SMS.
- Периодически перезагружайте телефон, затем открывайте SMS Relay и запускайте тест. После
  обновлений HiOS перепроверяйте Battery Lab: производитель может сбросить настройку.

## Сборка из исходников

Установите Android SDK 35, затем выполните:

```sh
cp local.properties.example local.properties
# Укажите корректный sdk.dir.
./gradlew clean lintDebug assembleDebug
```

Debug APK появится в `app/build/outputs/apk/debug/app-debug.apk`. Для стабильных обновлений
нужно хранить приватный ключ подписи и создать `signing.properties` по примеру
`signing.properties.example`:

```sh
./gradlew -PrelayDefaultUrl= -PrelayDefaultPin= clean lintRelease assembleRelease
```

Никогда не публикуйте ключ подписи, `signing.properties`, токен бота, номера получателей,
пароль сервера или приватные настройки по умолчанию.
